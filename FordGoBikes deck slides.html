1. Introduction

Dataset Overview:¶

FordGoBikes dataset includes a lot of features as it has more than 180,000 records of bikes trips and their information about individual rides made in a bike-sharing system covering the greater San Francisco Bay area in 2017, including stations coordinates, user types, birth year, gender, which can be easly interpret some insights.

Questions:¶

  • Which age subscribe the service often?
  • Which is the Busiest days in the week?
  • What is the Peak hours in the day?
  • Which Gender using the service the most?
  • What is the Average Number of Trips per Day?
  • What is the Average Number of Trips per Age?
  • Which Shares the bike the most?
  • Which are the most used Stations?
  • How long Trips take each Day?
  • What is Number of Trips and Distance per Age?

4. Conclusion

First, the data had a lot of Nulls which made it impossible to drop them all, so i had to replace the Null values in member_birth_year column with average birth year, also with member_gender we will replace them with 'Other' keyword, as there is a lot of records have much information so not be dropped, those two columns which contained most of the Nulls, the dataset contained no duplicated rows, small subset of the data contained invalid values, which had to be handled either by removing them or putting default value, as the age it had some records of ages more than 140, a little bit of Feature Engineering had to be done at first to create four columns Age, Trip_distance, Day_time, Day_week, Biker_speed to generate further more insights.

  • First thing i created visualizations to see the distrbution of ages as i saw the minimum value in that column is 1878 which translated to 143 years old, which seems there is outliers here, so i dropped ages more than 80 (203 rows).
  • Second i made a correlation matrix too see which columns has the highest correlation. helped in creating further more insights.
  • Thrid i created some insights to display some answers to the questions stated before, first one was two insights for Highest Peaks based on [Day_week, Day_time] which showed that Thursday and Wednesday were the highest peak of the trips, While Friday and Saturday were the lowest as they are the weekends in US. and Daily peak starts from 9 at morning and begin to decrease from afternoon, and then start to rise again from 3 and reaches the highst point at 5 PM.
  • Fourth insight made to see Which age uses the service often which showed that Most of the users are between ages 20-50 with an average of 37 is where the most of the users arround.
  • Fifth insight was made to illustrate the Stations scatter distribution which showed there are three spots which rides happens frequently with most frequency in the bottom spot.
  • Sixth insight was about Average Trip duration for each day which showed that Weekends is where the longest trips happens.
  • Seventh insight was about Gender bike share, as stated that Males portion in data is the highest while sharing is higher compared to males.
  • The Eighth insight here showed Subscribers regarding Age where Most subscribers lie between 26 to 37 years old.
  • in the Ninth insight we showed the Average Speed per Gender and as usual Males has the highest driving speed.

Limitations¶

First, the data had a lot of Nulls which made it impossible to drop them all, and made it harder to do the Data Preparation, as member_gender and member_birth_year had about 8265, and some columns had about 197 like end_station_id, but on the other side there were a critical issues that needs to be discussed:

  • Missing values I had to replace the Null values in member_birth_year column with average birth year, also with member_gender we will replace them with 'Other' keyword, as there is a lot of records have much information so not be dropped.

  • Invalid values Some of the ages had invalid values as some ages were over 100, so i had to drop them.

Overall the data did answer most of the questions, but for the most accurate anwer it needs to be enlarged and enhanced.
